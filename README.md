# 🚀 Frontend Component Challenge

Este teste tem como objetivo avaliar sua capacidade de desenvolver uma interface front-end moderna, organizada e escalável, seguindo boas práticas de mercado.

---

## 🎨 Layout

O layout base do projeto está disponível no Figma:

👉 https://www.figma.com/design/yO1Dobfgn4RzPLXjOwt1R3/Website-%7C-Lara?m=auto&t=6Z0HmDgS8pmlpaRG-6

**Importante:** O layout deve ser seguido com fidelidade (cores, tipografia, espaçamentos e proporções).

---

## 🧠 Objetivo

Mais do que reproduzir páginas, queremos avaliar sua capacidade de pensar em **arquitetura de componentes reutilizáveis**.

Evite estruturar o projeto com foco em páginas como "home" ou "sobre".
Priorize a construção de componentes como:

* Header
* Hero
* Cards
* Listagens
* Footer
* Blocos reutilizáveis

---

## 🧩 Por que isso é importante?

Trabalhamos com desenvolvimento de temas, especialmente em ambientes como WordPress, onde a **modularização é essencial**.

Os projetos não são construídos como páginas isoladas, mas sim como **conjuntos de componentes reutilizáveis**, que podem ser combinados de diferentes formas dependendo da necessidade.

Isso permite:

* Reutilização de código
* Maior escalabilidade
* Facilidade de manutenção
* Flexibilidade para montar diferentes layouts com os mesmos blocos

Por isso, queremos entender como você organiza e estrutura esses componentes no seu código.

---

## ⚙️ Requisitos técnicos

* Utilizar **HTML, CSS e JavaScript**
* Utilizar a biblioteca de slider: https://splidejs.com/
* Aplicar a metodologia **BEM CSS** para nomeação de classes
* Utilizar **SASS** para organização dos estilos
* Criar variáveis (principalmente para):

  * Cores
  * Tipografia
* Utilizar **REM** como unidade de medida padrão

---

## 📱 Boas práticas obrigatórias

* Mobile First
* HTML semântico
* CSS modularizado (separado por componentes e/ou páginas)
* SEO básico aplicado
* Acessibilidade (A11y)

---

## 🤖 Uso de IA

O uso de ferramentas de IA (como ChatGPT, Copilot, etc.) é **permitido**.

No entanto, esperamos que você:

* Entenda o código que está sendo utilizado
* Mantenha consistência e organização no projeto
* Tome decisões conscientes de implementação
